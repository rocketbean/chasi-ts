# Drizzle ORM + TypeScript Skill (chasi-ts)

## How Drizzle is wired in chasi-ts

The framework boots the Drizzle client from `config/database.ts`, loads the schema file, and exposes both the query client and table definitions through two access patterns in controllers.

### File locations

| File | Purpose |
|------|---------|
| `src/container/drizzle/schema.ts` | Table definitions — edit here to add/change tables |
| `drizzle/migrations/` | Generated SQL migration files — never edit manually |
| `drizzle.config.ts` | drizzle-kit config (dialect, schema path, migrations output) |
| `src/config/database.ts` → `pg` | Drizzle connection config |

### Connection config (`src/config/database.ts`)

```ts
pg: {
  driver: "drizzle",
  url: process.env.PG_URL,
  options: {
    adapter: "node-postgres",        // drizzle-orm sub-path to import
    schema: "./container/drizzle/schema",  // relative to dist/ after compile
    globals: { logger: false },      // passed directly to drizzle() constructor
  },
},
```

**Adapter → peer package** (install the client separately):

| `adapter` | Install |
|-----------|---------|
| `"node-postgres"` | `npm i pg @types/pg` |
| `"postgres-js"` | `npm i postgres` |
| `"mysql2"` | `npm i mysql2` |
| `"better-sqlite3"` | `npm i better-sqlite3 @types/better-sqlite3` |
| `"libsql"` | `npm i @libsql/client` |

For `mysql2` and `postgres-js`, pass a pre-built client via `globals.client` — the url field is ignored for those adapters:
```ts
globals: { client: await mysql.createConnection({ uri: process.env.MYSQL_URL }) }
```

## Schema (`src/container/drizzle/schema.ts`)

All table definitions are **named exports** — no default export for tables. The driver strips `default` and registers every other export as a table definition.

```ts
import {
  pgTable,
  uuid,
  varchar,
  timestamp,
  unique,
} from "drizzle-orm/pg-core";

export const properties = pgTable("properties", {
  id: uuid("id").primaryKey().defaultRandom(),
  name: varchar("name", { length: 255 }).notNull(),
  code: varchar("code", { length: 255 }).notNull(),
  createdAt: timestamp("created_at", { withTimezone: true }).notNull().defaultNow(),
  updatedAt: timestamp("updated_at", { withTimezone: true }).notNull().defaultNow(),
});

export const units = pgTable("units", {
  id: uuid("id").primaryKey().defaultRandom(),
  name: varchar("name", { length: 255 }).notNull(),
  code: varchar("code", { length: 255 }).notNull(),
  propertyId: uuid("property_id")
    .notNull()
    .references(() => properties.id, { onDelete: "cascade" }),
  createdAt: timestamp("created_at", { withTimezone: true }).notNull().defaultNow(),
  updatedAt: timestamp("updated_at", { withTimezone: true }).notNull().defaultNow(),
});
```

### Schema conventions in this project

- `uuid().primaryKey().defaultRandom()` — all PKs are UUIDs generated by the DB
- `timestamp("col_name", { withTimezone: true })` — always use `withTimezone`
- Column names are **snake_case** strings; TypeScript fields are **camelCase**
- Foreign keys always use `{ onDelete: "cascade" }` where the parent owns the child
- Multi-column unique constraints: `unique("constraint_name").on(table.col1, table.col2)` passed as a third argument to `pgTable`
- `updatedAt` is NOT auto-managed by Drizzle — update it manually or use a DB trigger

## Accessing Drizzle in controllers — two patterns

### Pattern 1: `Model.drizzle("connectionKey")` (recommended)

Returns the live Drizzle `db` client directly. Cleanest approach for queries.

```ts
import Controller from "../../../package/statics/Controller.js";
import Model from "../../../package/statics/Model.js";
import { properties, units } from "../../drizzle/schema.js";
import { eq } from "drizzle-orm";

export default class PropertyController extends Controller {
  get db() {
    return Model.drizzle("pg");  // "pg" = connection key in config/database.ts
  }

  async list(request, response) {
    return await this.db.select().from(properties);
  }

  async index(request, response) {
    const { id } = request.params;
    const [record] = await this.db
      .select()
      .from(properties)
      .where(eq(properties.id, id));
    if (!record) throw { status: 404, message: "Not found" };
    return record;
  }

  async create(request, response) {
    const [created] = await this.db
      .insert(properties)
      .values(request.body)
      .returning();
    return created;
  }

  async update(request, response) {
    const { id } = request.params;
    const [updated] = await this.db
      .update(properties)
      .set(request.body)
      .where(eq(properties.id, id))
      .returning();
    return updated;
  }

  async delete(request, response) {
    const { id } = request.params;
    await this.db.delete(properties).where(eq(properties.id, id));
    return { deleted: true };
  }
}
```

### Pattern 2: `this.models.connectionName` (via model collection)

The driver populates `this.models[connectionName]` with:
- `_db` — the live Drizzle query client (same as `Model.drizzle("pg")`)
- `[tableKey]` — each named export from the schema file

```ts
async list(request, response) {
  const { _db, properties } = this.models.pg;  // _db = query client, properties = table def
  return await _db.select().from(properties);
}
```

Use Pattern 1 for clarity. Pattern 2 is useful when you need table definitions without importing the schema file.

## drizzle.config.ts

Controls drizzle-kit (the migration CLI). Do not use this file for the runtime connection — that's in `config/database.ts`.

```ts
import "dotenv/config";
import { defineConfig } from "drizzle-kit";

export default defineConfig({
  dialect: "postgresql",
  schema: "./src/container/drizzle/schema.ts",  // TypeScript source (drizzle-kit reads TS directly)
  out: "./drizzle/migrations",
  dbCredentials: {
    url: process.env.PG_URL!,
  },
});
```

## Migrations (drizzle-kit)

```bash
# After editing src/container/drizzle/schema.ts:

# Generate a new SQL migration file (does NOT apply it)
npx drizzle-kit generate

# Apply all pending migrations to the database
npx drizzle-kit migrate

# Push schema changes directly without a migration file (dev only)
npx drizzle-kit push

# Open Drizzle Studio (GUI browser for the DB)
npx drizzle-kit studio
```

Migration files go to `drizzle/migrations/` and should be committed. Never edit them manually — regenerate instead.

## TypeScript types from schema

Drizzle infers types directly from the schema — no separate interface files needed.

```ts
import type { InferSelectModel, InferInsertModel } from "drizzle-orm";
import { properties, units } from "../../drizzle/schema.js";

type Property = InferSelectModel<typeof properties>;    // SELECT result shape
type NewProperty = InferInsertModel<typeof properties>; // INSERT input shape
```

Use these in controller method signatures:

```ts
async create(request, response) {
  const data: NewProperty = request.body;
  const [row] = await this.db.insert(properties).values(data).returning();
  return row;
}
```

## Query patterns

```ts
import { eq, and, or, like, gte, lte, isNull, desc, asc, sql } from "drizzle-orm";

const db = Model.drizzle("pg");

// Filter
await db.select().from(properties).where(eq(properties.name, "Sunset"));

// Compound conditions
await db.select().from(units).where(
  and(eq(units.propertyId, id), like(units.name, "%A%"))
);

// Join
await db
  .select({ property: properties, unit: units })
  .from(properties)
  .leftJoin(units, eq(units.propertyId, properties.id));

// Order + pagination
await db
  .select()
  .from(properties)
  .orderBy(desc(properties.createdAt))
  .limit(20)
  .offset(page * 20);

// Insert multiple rows
await db.insert(units).values([
  { name: "Unit A", code: "UA", propertyId: id },
  { name: "Unit B", code: "UB", propertyId: id },
]);

// Update with returning
const [updated] = await db
  .update(properties)
  .set({ name: "New Name", updatedAt: new Date() })
  .where(eq(properties.id, id))
  .returning();

// Transaction
await db.transaction(async (tx) => {
  const [prop] = await tx.insert(properties).values(propData).returning();
  await tx.insert(units).values(
    unitData.map((u) => ({ ...u, propertyId: prop.id }))
  );
  return prop;
});

// Raw SQL (escape hatch)
await db.execute(sql`SELECT pg_sleep(0)`);
```

## Adding a new table — checklist

1. Add the table definition to `src/container/drizzle/schema.ts` as a **named export**
2. Run `npx drizzle-kit generate` → creates a SQL file in `drizzle/migrations/`
3. Run `npx drizzle-kit migrate` → applies it to the DB
4. Import the table definition in your controller and use `Model.drizzle("pg")` to query
5. No registration needed — the driver picks up all named exports from the schema file automatically
